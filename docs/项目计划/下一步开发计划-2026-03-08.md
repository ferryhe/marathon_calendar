## 🎯 近期重点（2026年3月）

### 第1-2周（03-08 至 03-21）：数据质量与完善

#### 1. 数据去重与质量检查
- [ ] 实现数据去重逻辑
  - 基于赛事名称 + 日期 + 城市的相似度匹配
  - 智能合并重复数据（保留最完整信息）
  - 添加人工审核队列（管理后台）
- [ ] 建立数据质量检查
  - 必填字段验证（名称、日期、城市）
  - 日期合理性检查（不接受过去1年以上的日期）
  - 报名链接有效性检查（定期HTTP HEAD请求）

#### 2. 爬虫系统完善
- [ ] 测试并稳定现有3个爬虫
  - 最酷体育（zuicoolCrawler.ts）生产环境测试
  - 爱燃烧（iranshaoCrawler.ts）生产环境测试
  - 42trip（42tripCrawler.ts）生产环境测试
- [ ] 添加更多数据源（目标：8-10个）
  - 建议优先级：跑步圈、马拉松报名、大众体育
  - 参考：[研究报告-马拉松数据源调研.md](../研究报告/研究报告-马拉松数据源调研.md)
- [ ] 完善失败告警机制
  - 爬虫失败邮件/日志告警
  - 连续失败超过3次暂停并通知管理员

#### 3. 搜索功能增强
- [ ] 全文搜索支持（PostgreSQL全文检索）
- [ ] 拼音搜索支持（中文城市名、赛事名）
- [ ] 搜索结果高亮显示
- [ ] 搜索历史记录（本地存储）

### 第3-4周（03-22 至 04-04）：性能与安全

#### 4. 后端性能优化
- [ ] 添加Redis缓存
  - 赛事列表缓存（5分钟TTL）
  - 用户会话缓存优化
  - 热门赛事缓存
- [ ] 数据库查询优化
  - 添加复合索引（raceDate + status）
  - 优化分页查询（keyset pagination）
  - 分析慢查询日志

#### 5. API安全增强
- [ ] 实现API速率限制
  - 公开API：100次/分钟（基于IP）
  - 登录API：5次/分钟（防暴力破解）
  - 评论API：10次/分钟（防刷评）
- [ ] 添加请求日志
  - 记录关键操作（登录、发评论、管理操作）
  - 异常行为检测
