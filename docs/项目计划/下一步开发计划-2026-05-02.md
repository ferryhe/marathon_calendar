### P0 - 历史赛事自动归档 🔴 1h｜代码 + 数据修复 ✅ **已完成（2026-05-02b）**

**症状**：12 条赛事已经跑完了，前台还显示"报名中/即将开始"，看着就像没人维护。

**做法**：
1. 在后端加一个简单的 SQL：`UPDATE marathon_editions SET registration_status='已完赛' WHERE race_date < CURRENT_DATE - INTERVAL '1 day' AND registration_status NOT IN ('已完赛','已结束','已取消')`
2. 两个触发点：
   - `syncNowOnce()` 入口跑一次（已有 advisory lock，并发安全）
   - 加个 `/api/marathons/archive-past` 公开端点用于手动触发，复用 refresh 的 60s 限流
3. 立即手动跑一次 SQL 把当前 12 条修掉

**验收**：前台再无 race_date 已过但状态非已完赛/已结束/已取消的卡片。

### P1 - Zuicool 大批量补绑 🟡 2-3h｜数据脚本 ✅ **已完成（2026-05-02b，6 → 21）**

**目标**：把 Zuicool 上**已经存在**但我们没绑的国内 + 海外赛事一次性补齐。

**已知 Zuicool 上**有页面、我们尚未绑定的赛事（来自 /events 列表 + /events/newreg 抓取）：

国内大型城市马拉松（高优先级）：
- 北京、厦门、成都、武汉、苏州、宁波、大连、青岛、无锡、重庆、南京、西安、东营、衡水湖
- 即所有现存 28 个国内赛事中**没绑 Zuicool 的 22 个**，按 [研究报告/最酷zuicool爬取方案](../研究报告/研究报告-最酷zuicool爬取方案.md) 的"4 步发现流程"批量做

海外（Zuicool 海外栏目里直接列出的 7 个）：
- 波尔多红酒马、2027 东京（20 周年）、普吉、清迈、吉隆坡、香港、婆罗洲
- 注意：东京/巴黎/伦敦/纽约/柏林/芝加哥/波士顿 等 WMM 在 Zuicool 海外栏目**没出现**，可能是中国跑者通过国际旅行社走，不绑 Zuicool 也对

**做法**：
1. 写个 `script/discover-zuicool-bindings.ts`，输入现有赛事清单，对每个跑：
   - `webSearch({ queries: ['2026{城市}马拉松 site:zuicool.com'] })`
   - `webFetch` 验证 H1 届次正确（避开历史归档）
   - 输出 SQL upsert 增量
2. 人工 review 输出，确认无误后执行到 dev → prod
3. 文档落 `docs/数据库变更/2026-05-XX-zuicool-batch-binding.sql`

**验收**：Zuicool bindings 从 6 → 25+。
