## 🎯 核心建议摘要

### 数据采集方案
```
推荐：分层策略
- 第一层：20-30个知名官网 + 2-3个报名平台
- 第二层：省市级官网 + 其他平台
- 第三层：搜索引擎 + 用户提交
```

### 数据提取方案
```
推荐：混合方案
- 90%：传统解析（Cheerio/Puppeteer）
- 8%：AI辅助（传统解析失败时）
- 2%：人工审核（AI不确定时）

AI选择：阿里云通义千问
成本：约¥30-60/月
```

### 开发路线
```
优先级：Web版 → 小程序 → 统一后台 → 高级功能

理由：
1. Web版开发快，可快速验证
2. 后端API两端共用
3. 小程序依赖成熟的后端支持
```

### 技术栈
```
前端：React + Vite + Tailwind CSS
后端：Express + PostgreSQL + Redis
爬虫：Puppeteer + 通义千问
云服务：腾讯云
```

### 预算
```
运营成本：¥690-1220/月
- 云服务器：¥200-300
- 云数据库：¥300-500
- Redis：¥100-200
- CDN+存储：¥50-100
- AI API：¥30-100
```
